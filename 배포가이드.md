# 1단계: Git 준비 (처음이면)

## 1-1. Git 설치 확인

PowerShell이나 명령 프롬프트에서:
```bash
git --version
```
`git version 2.xx.x` 비슷한 게 나오면 설치된 거예요.

안 나오면: https://git-scm.com/download/win 에서 다운로드해 설치하세요. 설치 중 옵션은 **전부 기본값(Next)**으로 두면 됩니다.

## 1-2. Git 사용자 정보 등록 (최초 1회만)

```bash
git config --global user.name "조국형"
git config --global user.email "candj0818@gmail.com"
```

# 2단계: GitHub에 저장소 만들기

## 2-1. 저장소 생성

1. https://github.com/new 접속
2. 아래처럼 입력:
   - **Repository name**: `gongsi-lookup` (원하는 이름)
   - **Description**: `경매 주소로 공시가격을 조회하는 웹사이트` (선택)
   - **Public** 선택 ← Vercel 무료 플랜은 Public 저장소가 편해요
   - **아래 체크박스는 전부 끄기** (README, .gitignore, license ❌)
3. **Create repository** 클릭

## 2-2. 생성 직후 화면에 나오는 주소 복사

생성하면 이런 주소가 나와요:
```
https://github.com/내아이디/gongsi-lookup.git
```
이 주소를 복사해두세요. 곧 씁니다.

# 3단계: 코드를 GitHub에 올리기

PowerShell에서 **현재 프로젝트 폴더로 이동**:
```bash
cd "C:\Users\asusbook-cho\Desktop\클로드 경매주소공시지가"
```

그다음 아래 명령을 **순서대로** 실행:

```bash
# 이 폴더를 git 저장소로 초기화
git init

# .env.local이 제외됐는지 확인 (빨간색 리스트에 .env.local 나오면 안 됨)
git status

# 모든 파일 추가 (.gitignore 덕분에 node_modules, .env.local 제외됨)
git add .

# 첫 커밋
git commit -m "초기 버전: 공시가격 간편 조회 웹사이트"

# 기본 브랜치를 main으로
git branch -M main

# GitHub 저장소 연결 (아까 복사한 주소를 붙여넣기)
git remote add origin https://github.com/내아이디/gongsi-lookup.git

# 업로드
git push -u origin main
```

**처음 push할 때** GitHub 로그인 창이 뜰 수 있어요. 브라우저로 로그인하면 됩니다.

✅ `git push` 성공 후 GitHub에서 저장소를 새로고침하면 파일들이 올라와 있어야 해요.

⚠️ **반드시 확인**: 저장소 파일 목록에 `.env.local`이 **없어야** 합니다. 있으면 API 키가 노출된 거예요. (지금 .gitignore가 막아주고 있어서 문제없습니다)

# 4단계: Vercel에 배포

## 4-1. Vercel 가입

1. https://vercel.com 접속
2. 우상단 **Sign Up** → **Continue with GitHub** 선택
3. GitHub 권한 승인

## 4-2. 저장소 Import

1. Vercel 대시보드 → **Add New...** → **Project**
2. 방금 만든 `gongsi-lookup` 저장소 옆의 **Import** 클릭
3. 설정 화면 나옴:
   - **Project Name**: 자동 입력됨 (원하면 변경 — 이게 URL에 들어감)
   - **Framework Preset**: `Other` 로 자동 선택됨 ✅
   - **Root Directory**: `./` 그대로
   - **Build Command**: 빈칸 또는 자동
   - **Output Directory**: 빈칸 또는 자동
4. **Environment Variables** 섹션을 펼쳐서 아래 3개 추가:

| Key | Value |
|---|---|
| `DATA_GO_KR_KEY` | `D68A5D88-C3E9-39F3-A4A8-DA16037F81B1` |
| `VWORLD_KEY` | `D68A5D88-C3E9-39F3-A4A8-DA16037F81B1` |
| `MOCK_MODE` | `false` |

5. **Deploy** 클릭 → 1~2분 기다리면 완료 🎉

## 4-3. 배포된 주소 확인

완료되면 이런 주소가 나와요:
```
https://gongsi-lookup.vercel.app
(또는 https://gongsi-lookup-xxxxxxx.vercel.app 형태)
```

아직 열지 마세요 — 다음 단계에서 VWorld 키에 이 도메인을 등록해야 합니다.

# 5단계: VWorld 키에 배포 도메인 등록 (⚠️ 필수!)

지금 VWorld 키는 `localhost:3000`에만 허용되어 있어요. 배포된 도메인에서 API가 작동하려면 그 도메인을 키에 추가해야 합니다.

## 5-1. VWorld 로그인

1. https://www.vworld.kr/ 접속 → 로그인
2. 우상단 프로필 → **마이페이지** 또는 **오픈API 인증키 관리**

## 5-2. 인증키에 도메인 추가

1. 발급받은 키 목록에서 해당 키의 **수정** 클릭
2. **승인된 도메인** 또는 **서비스 URL** 필드에 다음 두 줄 **둘 다** 추가:
   ```
   http://localhost:3000
   https://gongsi-lookup.vercel.app
   ```
   (여러분의 실제 Vercel 주소를 넣으세요)
3. 저장

💡 **팁**: Vercel은 `preview` 배포마다 URL이 바뀔 수 있어요. 처음엔 메인 도메인(`https://gongsi-lookup.vercel.app`)만 등록하고 문제 생기면 더 추가하세요.

# 🔧 문제 해결

### ❌ "주소의 법정동코드를 찾지 못했습니다"
- 지오코딩 API 호출 실패. VWorld 키 도메인 등록 여부 재확인
- Vercel 대시보드 → 프로젝트 → **Logs** 탭에서 에러 메시지 확인

### ❌ 조회 버튼 눌러도 아무 반응 없음
- 브라우저 개발자도구(F12) → Console 탭에서 에러 확인
- Network 탭에서 `/api/lookup` 요청의 응답 확인

### ❌ Vercel 빌드가 실패
- Vercel 대시보드에서 빌드 로그 확인
- 대부분 환경변수 누락 문제 (4-2 단계 확인)

### ❌ 로컬에선 되는데 배포만 안 됨
- 십중팔구 VWorld 도메인 등록 문제. 5단계 재확인
- 환경변수 값에 **앞뒤 공백** 들어갔는지 확인
